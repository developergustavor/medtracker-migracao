A) Tab Separada (Atual Simplificado)

Mantém o step/tab "Submateriais" separado do form de dados. Dentro do tab: form inline compacto (nome + qtd + botão add) no topo, lista dos submateriais abaixo com remove (x). Sem tabela complexa — apenas lista simples numerada.

Pros

  • Separação clara: dados vs composição
  • Muito espaço para a lista de itens
  • UX familiar (similar ao original)

Cons

  • Precisa trocar de tab
  • Não vê os dados do KIT enquanto adiciona

B) Seção Inline "Composição do Kit"

Sem tab separada. Quando tipo=KIT, aparece uma seção destacada (borda primary sutil) dentro do mesmo form com: input inline para adicionar (nome + qtd + botão), lista com contadores +/- por item e remove (x). Tudo no mesmo scroll, sem troca de contexto.

Pros

  • Zero troca de contexto — tudo em uma tela
  • Vê dados + composição ao mesmo tempo
  • Contador +/- inline = edição rápida de quantidades
  • Seção só aparece para KIT

Cons

  • Form fica mais longo (scroll)
  • Com muitos submateriais a lista pode ficar grande

C) Painel Lateral (junto com imagens)

Mesmo padrão da coluna de imagens — form à esquerda, painel de composição do kit à direita (substitui a coluna de imagens quando tipo=KIT, ou fica abaixo dela). Input compacto no topo do painel para adicionar, lista com remove. Qtd submateriais auto-calculada.

Pros

  • Consistente com coluna lateral de imagens
  • Dados + composição sempre visíveis
  • Sem scroll extra

Cons

  • Painel lateral pode ficar apertado
  • Em KIT com imagens, precisa empilhar imagens + composição
