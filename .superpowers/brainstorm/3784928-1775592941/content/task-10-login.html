Funcionalidades

  • 4 abas: Codigo, Usuario (CPF), CME, Facial (placeholder). Segmented control com transicao suave.
  • Aba Codigo: Campo password, icone QR. API: POST /api/auth action user-by-code
  • Aba Usuario: CPF (mascara ###.###.###-##) + Senha (toggle visibilidade). API: POST /api/auth action user
  • Aba CME: Usuario + Senha. API: POST /api/auth action cme
  • Aba Facial: Placeholder — icone + "Funcionalidade futura" + botao desabilitado
  • 2FA: Se user.auth2FA && emailChecked, troca para view de OTP (5 digitos, inputs individuais). Email mascarado. Reenviar/Voltar
  • Remember Me: Checkbox, salva no localStorage
  • Esqueci minha senha: Link para /recuperacao-senha
  • Acessar disabled: Ate campos obrigatorios preenchidos + validacao Zod
  • Erro: Toast/alert com mensagem do backend (usa design system tokens)
  • Sucesso: store.login() + redirect /home (ou /dashboard-cme para CME)
  • Validacao: Zod schemas ja existentes (loginByCodeSchema, loginByCpfSchema, loginByCmeSchema)
  • React Hook Form: Para gerenciamento de formulario
  • Preloader: Botao mostra spinner durante request
  • Dark/Light: Segue tema do sistema ou ultimo salvo
  • Mobile: Full-width sem card wrapper, padding lateral
  • Logo: Usa logo-full-gradient.svg (light) / logo-white-full-gradient.svg (dark)
